实现微调流程中的 Stage 1：冻结的 OccAny 3D Reconstruction 分支，以及可训练的 reconstruction lifting / aggregation 分支。

输入是一个 batch 的 5 帧单目前视图像：
images: [B, 5, 3, H_img, W_img]

所有 OccAny 参数都必须被冻结。只运行 OccAny 的 reconstruction stage。Reconstruction encoder E 和 reconstruction decoder D 都不可训练。第一帧被视为 reference frame。需要从 reconstruction decoder 中提取每一帧对应的以下输出：
1. reconstruction decoder patch tokens T_rec: [B, N_rec, H_t, W_t, C_token]
2. global pointmaps P_rec_global: [B, N_rec, H_p, W_p, 3]
3. confidence maps C_rec: [B, N_rec, H_p, W_p]
4. 可选：predicted poses / trajectory 和 scene memory M，但 Stage 1 lifting 只需要 T_rec、P_rec_global 和 C_rec。

注意：T_rec 是 patch-level 的低分辨率特征，而 P_rec_global 和 C_rec 是 pixel-level 的高分辨率输出。不要把一个 patch token 直接投影到一个 3D 点上。正确做法是先把 T_rec 从 [H_t, W_t] 上采样到 pointmap 的分辨率 [H_p, W_p]，使每个像素位置 (u, v) 都有一个与之对齐的特征 T_rec_up(u, v)。第一版实现中使用 nearest-neighbor upsampling，以保持 patch token 的身份不被混合。

上采样之后，加入 pixel coordinate encoding。对于每个像素 (u, v)，计算一个像素坐标编码 e_pix(u, v)，例如归一化到 [-1, 1] 的坐标，或者相对于原始 patch cell 的局部坐标。然后将上采样后的 token 和像素坐标编码拼接：
x_rec(u, v) = concat(T_rec_up(u, v), e_pix(u, v))

将 x_rec(u, v) 输入一个可训练的 pixel feature MLP / token adapter，得到最终的 pixel feature：
f_rec(u, v) = MLP_rec(x_rec(u, v))
输出特征维度应为 C_lift。

然后使用 pixel-level global pointmap 将这些 pixel features lift 到目标 occupancy voxel grid 中。对于每个像素，P_rec_global(u, v) 给出了它在 OccAny reference camera 坐标系下的 3D 位置。在 voxelization 之前，需要使用给定的刚体变换 T_target_from_refcam，将 P_rec_global 转换到 occupancy target 坐标系中，例如 SemanticKITTI 的 LiDAR / ego 坐标系。

对于每个转换后的 3D 点 p = (x, y, z)，计算它的 voxel index：
i = floor((x - x_min) / voxel_size_x)
j = floor((y - y_min) / voxel_size_y)
k = floor((z - z_min) / voxel_size_z)
丢弃所有落在目标 voxel grid 外部的点。

多个像素可能会被投影到同一个 voxel 中。此时使用 confidence-weighted mean pooling 进行聚合。对于每个 voxel q，收集所有落入 q 的有效像素：
S_q = {pixel r | voxel(P_rec_global(r)) = q}
使用 OccAny 的 confidence value 作为权重：
w_r = C_rec(r)
然后计算：
V_rec(q) = sum_{r in S_q} w_r * f_rec(r) / (sum_{r in S_q} w_r + eps)
同时计算 weight / coverage volume：
W_rec(q) = sum_{r in S_q} w_r

返回：
V_rec: [B, C_lift, X, Y, Z]
W_rec: [B, 1, X, Y, Z]

不要在这个 stage 中混合 reconstruction 和 rendering features。Stage 1 只负责产生 V_rec 和 W_rec。Rendering branch 会单独产生 V_rnd 和 W_rnd，之后才会在 voxel space 中融合 [V_rec, W_rec, V_rnd, W_rnd]。

实现要求：
- OccAny 必须在 torch.no_grad() 下运行，或者所有 OccAny 输出都必须 detach。
- 只有 token adapter / pixel feature MLP 以及后续 lifting 相关的可训练层需要梯度。
- lifting 操作可以通过对 flatten 后的 pixels 使用 scatter_add 来实现。
- 使用 confidence-weighted mean aggregation，不要覆盖，不要随机选择，也不要直接 raw sum。
- 保留 W_rec 作为显式输出，因为后续 voxel-space fusion 需要知道哪些 voxel 有较强的观测支持。
